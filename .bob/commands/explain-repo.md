# /explain-repo - AI-Powered Repository Documentation Generator

## Description
Analyzes your entire codebase and generates a comprehensive "Repo Bible" - a complete guide that helps new developers understand and contribute to the project in under 30 minutes.

## How It Works
When you run `/explain-repo`, Bob will:
1. Scan the complete repository structure and file organization
2. Intelligently identify and read the most critical files (entry points, configs, core logic)
3. Detect the tech stack, architecture patterns, and data flows
4. Generate a detailed "Repo Bible" documentation
5. Auto-save the documentation to `output/repo-explainer.md`
6. Display a summary with key insights and complexity warnings

## Prerequisites
- Run this command from the root of your project directory
- Works best with projects that have clear structure and documentation
- No git repository required (works with any codebase)

## Instructions for Bob

You are now acting as a **Senior Software Architect** tasked with creating comprehensive onboarding documentation for a new developer joining the team. Your goal is to make the codebase completely understandable in under 30 minutes of reading.

### Step 1: Scan Repository Structure
First, get a complete overview of the project:
```bash
# List all directories and files recursively
ls -R
# Or use tree if available
tree -L 3 -I 'node_modules|venv|.git|dist|build'
```

Use the `list_files` tool recursively to understand the project structure.

### Step 2: Identify Critical Files
Prioritize reading these files (in order of importance):
1. **Project Definition Files**: `package.json`, `requirements.txt`, `Cargo.toml`, `go.mod`, `pom.xml`, `build.gradle`
2. **Documentation**: `README.md`, `CONTRIBUTING.md`, `docs/*`
3. **Configuration**: `.env.example`, `config/*`, `*.config.js`, `*.yaml`, `docker-compose.yml`
4. **Entry Points**: `index.js`, `main.py`, `main.go`, `app.js`, `server.js`, `src/index.*`
5. **Core Business Logic**: Files in `src/`, `lib/`, `app/`, `core/`
6. **Database/Models**: `models/*`, `schema/*`, `migrations/*`
7. **API Routes**: `routes/*`, `controllers/*`, `api/*`
8. **Tests**: `test/*`, `__tests__/*`, `*.test.*`, `*.spec.*`

**Smart Filtering**: Read up to 30-40 most important files. Skip:
- `node_modules/`, `venv/`, `.git/`, `dist/`, `build/`, `coverage/`
- Binary files, images, videos
- Generated files, lock files
- Vendor/third-party code

### Step 3: Analyze Architecture
For each file you read, identify:
- **Purpose**: What problem does this file solve?
- **Dependencies**: What does it import/require?
- **Exports**: What does it expose to other parts?
- **Patterns**: What design patterns are used?
- **Complexity**: How difficult is this code to understand?

### Step 4: Map Data Flow
Trace how data moves through the application:
- User input → Validation → Processing → Storage → Response
- External API calls and their purposes
- Database queries and data transformations
- Event flows and message passing
