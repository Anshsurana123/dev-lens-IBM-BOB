# /performance-check - AI-Powered Performance Analyzer

## Description
Analyzes your codebase for performance bottlenecks, inefficient algorithms, memory leaks, and optimization opportunities. Provides specific recommendations with benchmarks and optimized code examples.

## How It Works
When you run `/performance-check`, Bob will:
1. Scan code for performance anti-patterns
2. Identify inefficient algorithms (O(n²), O(n³))
3. Detect N+1 query problems
4. Find memory leaks and resource issues
5. Analyze database query efficiency
6. Check for blocking operations
7. Suggest optimizations with code examples
8. Auto-save the report to `output/performance-report.md`

### Algorithm Complexity Issues

#### Nested Loops (O(n²))
**Location**: `[filename]` (Line [X])

**Current Implementation**:
```[language]
for (let i = 0; i < array1.length; i++) {
  for (let j = 0; j < array2.length; j++) {
    // O(n²) operation
  }
}
```

**Optimized Implementation**:
```[language]
// Use Map for O(n) lookup
const map = new Map(array2.map(item => [item.id, item]));
for (let item of array1) {
  const match = map.get(item.id); // O(1) lookup
}
```

**Performance Gain**: O(n²) → O(n)

#### N+1 Query Problem
**Location**: `[filename]` (Line [X])

**Current Implementation**:
```[language]
// Fetches users one by one (N+1 queries)
const users = await User.findAll();
for (let user of users) {
  user.posts = await Post.findAll({ where: { userId: user.id } });

```

**Optimized Implementation**:

## 🔬 Profiling Guide

### How to Profile Your Application

#### Node.js
```bash
# Generate CPU profile
node --prof app.js

# Analyze profile
node --prof-process isolate-*.log > profile.txt

# Use clinic.js for detailed analysis
clinic doctor -- node app.js
```

#### Python
```python
import cProfile
import pstats

# Profile your code
cProfile.run('your_function()', 'output.prof')

# Analyze results
stats = pstats.Stats('output.prof')
stats.sort_stats('cumulative')
stats.print_stats(10)
```

#### Java
```bash
# Enable JFR (Java Flight Recorder)
java -XX:StartFlightRecording=duration=60s,filename=recording.jfr -jar app.jar

# Analyze with JMC (Java Mission Control)
```

## ⚠️ Common Performance Anti-Patterns

### 1. Premature Optimization
❌ **Don't**: Optimize before measuring
✅ **Do**: Profile first, then optimize

### 2. Ignoring Big-O Complexity
❌ **Don't**: Use nested loops for large datasets
✅ **Do**: Use appropriate data structures (Map, Set)

### 3. Not Using Indexes
❌ **Don't**: Query without indexes
✅ **Do**: Add indexes on frequently queried columns

### 4. Loading Everything at Once
❌ **Don't**: Load entire dataset into memory
✅ **Do**: Use pagination and streaming

### 5. Synchronous Operations
❌ **Don't**: Block the event loop
✅ **Do**: Use async operations
