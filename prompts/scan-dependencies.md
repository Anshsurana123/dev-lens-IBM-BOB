# /scan-dependencies - Dependency Vulnerability Scanner

## Description
Scans your project dependencies for known security vulnerabilities, outdated packages, and license compliance issues. Provides actionable recommendations for updates and security patches.

## How It Works
When you run `/scan-dependencies`, Bob will:
1. Detect package manager (npm, pip, maven, etc.)
2. Read dependency files (package.json, requirements.txt, etc.)
3. Check for known vulnerabilities (CVEs)
4. Identify outdated packages
5. Analyze license compatibility
6. Suggest safe update paths
7. Auto-save the report to `output/dependency-scan.md`

## Usage Examples

```bash
# Scan all dependencies
/scan-dependencies

# Scan specific package manager
/scan-dependencies --npm
/scan-dependencies --pip
/scan-dependencies --maven

# Check for critical vulnerabilities only
/scan-dependencies --critical-only
```

## 🛠️ Fix Commands

### Quick Fix (Automated)
```bash
# Fix all vulnerabilities automatically
npm audit fix

# Fix including breaking changes
npm audit fix --force

# For Python
pip install --upgrade -r requirements.txt

# For Java (Maven)
mvn versions:use-latest-versions
```

### Manual Fix (Recommended)
```bash
# Update specific packages
npm install [package1]@[version]
npm install [package2]@[version]

# Verify no breaking changes
npm test

# Commit the updates
git add package.json package-lock.json
git commit -m "fix(deps): update dependencies to fix security vulnerabilities"
```

## 🎯 Best Practices

### Dependency Management
1. **Keep dependencies up to date**: Update regularly, not just when vulnerabilities are found
2. **Minimize dependencies**: Fewer dependencies = smaller attack surface
3. **Pin versions**: Use exact versions in production
4. **Review before adding**: Check package health before adding new dependencies
5. **Automate scanning**: Set up CI/CD to scan on every commit

### Security Practices
1. **Use lock files**: Ensure consistent installs
2. **Audit regularly**: Run security scans weekly
3. **Monitor advisories**: Subscribe to security mailing lists
4. **Have update policy**: Define when and how to update
5. **Test thoroughly**: Always test after updates

## Automation Tips

### Set Up Automated Scanning

**GitHub Actions**:
```yaml
name: Dependency Scan
on: [push, pull_request]
jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - run: npm audit
```

**Pre-commit Hook**:
```bash
#!/bin/bash
npm audit --audit-level=high
```

### Continuous Monitoring

1. **Dependabot**: Enable in GitHub repository settings
2. **Snyk**: Integrate with your CI/CD pipeline
3. **npm audit**: Run in CI on every commit
4. **DevLens**: Run `/scan-dependencies` weekly
