## Analysis Guidelines

When analyzing the repository, please:

1. **Be Comprehensive**: Cover all aspects of the codebase
2. **Be Clear**: Use plain English, avoid jargon when possible
3. **Be Specific**: Reference actual files, functions, and patterns
4. **Be Practical**: Focus on information that helps developers
5. **Be Honest**: Note complexity, technical debt, and risks
