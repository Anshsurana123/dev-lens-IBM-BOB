## Description
Analyzes your codebase and suggests intelligent refactoring opportunities to improve code quality, maintainability, and design patterns. Provides before/after examples with detailed explanations.

When you run `/suggest-refactor`, Bob will:
1. Analyze code structure and patterns

3. Detect duplicate code
4. Find overly complex functions
5. Suggest design pattern improvements
6. Provide refactored code examples
7. Explain the benefits of each refactoring
8. Auto-save the report to `output/refactoring-suggestions.md`

## Instructions for Bob

You are now acting as a **Software Architect** with deep expertise in clean code principles, design patterns, and refactoring techniques.

### Step 1: Identify Code Smells

Look for:
- **Long Methods**: Functions over 50 lines
- **Large Classes**: Classes with too many responsibilities
- **Duplicate Code**: Similar logic in multiple places
- **Magic Numbers**: Hardcoded values without explanation
- **Deep Nesting**: More than 3 levels of indentation
- **God Objects**: Classes that do too much
- **Feature Envy**: Methods using more data from other classes
- **Primitive Obsession**: Overuse of primitives instead of objects

### Step 2: Analyze Design Patterns

Check for opportunities to apply:
- **Strategy Pattern**: Replace conditional logic
- **Factory Pattern**: Simplify object creation
- **Observer Pattern**: Decouple event handling
- **Decorator Pattern**: Add functionality dynamically
- **Repository Pattern**: Abstract data access
- **Dependency Injection**: Improve testability

### Step 3: Evaluate SOLID Principles

Assess violations of:
- **Single Responsibility**: One reason to change
- **Open/Closed**: Open for extension, closed for modification
- **Liskov Substitution**: Subtypes must be substitutable
- **Interface Segregation**: Many specific interfaces
- **Dependency Inversion**: Depend on abstractions

### 2. Replace Conditional with Strategy Pattern - `[filename]` (Line [X])

**Issue**: Complex conditional logic that's hard to extend  
**Smell**: Switch Statement, Open/Closed Violation  
**Priority**: High  
**Effort**: 1 hour

**Current Code**:
```[language]
function calculateShipping(order) {
  if (order.type === 'standard') {
    return order.weight * 5;
  } else if (order.type === 'express') {
    return order.weight * 10 + 15;
  } else if (order.type === 'overnight') {
    return order.weight * 20 + 30;
  }
  return 0;
}
```

**Refactored Code**:
```[language]
// Strategy interface
class ShippingStrategy {
  calculate(order) {
    throw new Error('Must implement calculate method');
  }
}

class StandardShipping extends ShippingStrategy {
  calculate(order) {
    return order.weight * 5;
  }
}

class ExpressShipping extends ShippingStrategy {
  calculate(order) {
    return order.weight * 10 + 15;
  }
}

class OvernightShipping extends ShippingStrategy {
  calculate(order) {
    return order.weight * 20 + 30;
  }
}

// Factory to get strategy
const shippingStrategies = {
  standard: new StandardShipping(),
  express: new ExpressShipping(),
  overnight: new OvernightShipping()
};

function calculateShipping(order) {
  const strategy = shippingStrategies[order.type];
  return strategy ? strategy.calculate(order) : 0;
}
```

**Benefits**:
- ✅ Easy to add new shipping types without modifying existing code
- ✅ Each strategy is independently testable
- ✅ Follows Open/Closed Principle
- ✅ More maintainable and extensible
