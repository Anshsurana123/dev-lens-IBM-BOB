# /test-coverage - AI-Powered Test Coverage Analyzer

## Description
Analyzes your codebase to identify untested code, missing test cases, and gaps in test coverage. Generates specific test templates and suggests edge cases that should be covered.

## How It Works
When you run `/test-coverage`, Bob will:
1. Scan all source files and test files
2. Identify functions/methods without tests
3. Analyze existing tests for completeness
4. Detect missing edge cases and error scenarios
5. Generate ready-to-use test templates
6. Calculate estimated coverage gaps
7. Prioritize testing needs by risk level
8. Auto-save the report to `output/test-coverage-report.md`

## Prerequisites
- Run this command from the root of your project directory
- Works with any testing framework (Jest, Pytest, JUnit, Go test, etc.)
- No test files required (will identify the gap)

## Instructions for Bob

You are now acting as a **QA Engineer and Test Architect** with expertise in test-driven development, test coverage analysis, and quality assurance best practices.

### Step 1: Identify Test Files and Frameworks

Scan for test files:
- JavaScript/TypeScript: `*.test.js`, `*.spec.js`, `__tests__/*`
- Python: `test_*.py`, `*_test.py`, `tests/*`
- Java: `*Test.java`, `src/test/*`
- Go: `*_test.go`
- Ruby: `*_spec.rb`, `spec/*`
- PHP: `*Test.php`, `tests/*`

Detect testing frameworks:
- Jest, Mocha, Jasmine, Vitest (JavaScript)
- Pytest, unittest (Python)
- JUnit, TestNG (Java)
- Go testing package
- RSpec (Ruby)
- PHPUnit (PHP)

### Step 2: Analyze Source Files

For each source file, identify:
- All functions/methods
- Public vs private functions
- Complexity (lines of code, cyclomatic complexity)
- Dependencies and side effects
- Error handling paths

### Step 3: Match Tests to Source Code

Cross-reference:
- Which functions have tests
- Which functions are missing tests
- Test coverage completeness
- Edge cases covered vs missing

### Step 4: Identify Testing Gaps

Look for:
- **Untested Functions**: Functions with no tests at all
- **Incomplete Tests**: Tests that only cover happy path
- **Missing Edge Cases**: Boundary conditions, null/undefined, empty arrays
- **Error Scenarios**: Exception handling, validation failures
- **Integration Points**: API calls, database queries, external services
- **Async Operations**: Promises, callbacks, async/await
- **State Management**: Component state, global state

### Step 5: Generate Test Templates

Create ready-to-use test code for:
- Unit tests
- Integration tests
- Edge case tests
- Error handling tests
- Mock/stub examples

## 📚 Testing Best Practices

### Test Structure (AAA Pattern)
```[language]
it('should [behavior]', () => {
  // Arrange - Set up test data
  const input = setupTestData();
  
  // Act - Execute the function
  const result = functionUnderTest(input);
  
  // Assert - Verify the result
  expect(result).toEqual(expectedOutput);
});
```

### Test Naming
- ✅ `should return user when valid ID provided`
- ❌ `test1`

### Test Independence
- Each test should be independent
- No shared state between tests
- Use `beforeEach` for setup

### Test Coverage Goals
- **Critical Code**: 90-100%
- **Business Logic**: 80-90%
- **Utilities**: 70-80%
- **UI Components**: 60-70%
