## Key Files Explained

### Entry Points
- **src/index.js**: Main entry point, starts the server
- **src/app.js**: Express app configuration with middleware
- **src/server.js**: HTTP server setup with graceful shutdown

### Configuration
- **src/config/database.js**: PostgreSQL connection using Sequelize ORM
- **src/config/redis.js**: Redis connection for caching and sessions
- **src/config/jwt.js**: JWT token configuration and secrets

### Data Layer
- **src/models/**: Sequelize models defining database schema
- **src/db/migrations/**: Database schema migrations
- **src/db/seeders/**: Sample data for development

### Business Logic
- **src/controllers/**: Handle HTTP requests, call services
- **src/services/**: Core business logic, reusable across controllers
- **src/middleware/**: Request processing (auth, validation, logging)

### API Layer
- **src/routes/**: Express route definitions
- **src/utils/validators.js**: Input validation schemas (Joi)

### External Integrations
- **src/services/paymentService.js**: Stripe payment processing
- **src/services/emailService.js**: SendGrid email sending
- **src/services/searchService.js**: Elasticsearch product search

### Testing
- **tests/unit/**: Unit tests for individual functions
- **tests/integration/**: API endpoint integration tests
- **tests/fixtures/**: Test data

## Data Flow Example

### User Places an Order

1. **Client** → POST /api/orders
2. **routes/orders.js** → Receives request
3. **middleware/auth.js** → Verifies JWT token
4. **middleware/validation.js** → Validates order data
5. **controllers/orderController.js** → Handles request
6. **services/inventoryService.js** → Checks product availability
7. **services/paymentService.js** → Processes payment via Stripe
8. **models/Order.js** → Creates order in database
9. **models/OrderItem.js** → Creates order items
10. **services/emailService.js** → Sends confirmation email
11. **cacheService.js** → Invalidates user's order cache
12. **Response** → Returns order details to client

## Architecture Pattern

**Layered Architecture (MVC + Services)**

```
┌─────────────────────────────────────────┐
│           Client (Browser/App)          │
└──────────────────┬──────────────────────┘
                   │ HTTP/HTTPS
                   ▼
┌─────────────────────────────────────────┐
│         Routes (API Endpoints)          │
│  /api/auth, /api/products, /api/orders  │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│      Middleware (Auth, Validation)      │
│   JWT Verify, Rate Limit, CORS, etc.   │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│      Controllers (Request Handlers)     │
│  Parse request, call services, respond  │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│     Services (Business Logic Layer)     │
│  Auth, Payment, Email, Inventory, etc.  │
└──────────────────┬──────────────────────┘
                   │
        ┌──────────┴──────────┐
        ▼                     ▼
┌──────────────┐      ┌──────────────┐
│   Models     │      │   External   │
│  (Database)  │      │   Services   │
│  Sequelize   │      │ Stripe, etc. │
└──────┬───────┘      └──────────────┘
       │
       ▼
┌──────────────┐
│  PostgreSQL  │
│   Database   │
└──────────────┘
```

## Complexity Warnings

### High Complexity Areas
1. **src/services/paymentService.js** - Complex Stripe integration with webhooks
2. **src/controllers/orderController.js** - Multi-step order processing logic
3. **src/models/index.js** - Complex model associations and relationships

### Common Pitfalls
1. **N+1 Queries**: Watch for eager loading in product listings
2. **Race Conditions**: Inventory checks need proper locking
3. **Memory Leaks**: Redis connections must be properly closed

## Getting Started

```bash
# Install dependencies
npm install

# Copy environment variables
cp .env.example .env

# Start PostgreSQL and Redis with Docker
docker-compose up -d

# Run migrations
npm run migrate

# Seed database with sample data
npm run seed

# Start development server
npm run dev

# Server runs at http://localhost:3000
```

## API Endpoints

- **POST /api/auth/register** - Register new user
- **POST /api/auth/login** - Login user
- **GET /api/products** - List products (with pagination)
- **GET /api/products/:id** - Get product details
- **POST /api/orders** - Create new order (requires auth)
- **GET /api/orders/:id** - Get order details (requires auth)
- **GET /api/users/me** - Get current user (requires auth)
